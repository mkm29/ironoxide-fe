What's in the image

Three things that don't normally play nicely together.

01 · microVM

Hardened Linux microVM

krunkit + libkrun. Boots from a single golden image. No installer, no orchestrator, no daemon.

  • Kernellibkrun
  • Imageironoxide.raw
  • Cold boot~800ms
02 · GPU

GPU passthrough

Vulkan inside the guest, Metal on the host. Venus + virglrenderer serialize the trust boundary.

  • Guest APIVulkan
  • Host APIMetal
  • BridgeVenus
03 · Agent

Coding agent included

OpenCode talks HTTP to llama-server. Drop-in OPENAI_BASE_URL. Bring your own editor.

  • AgentOpenCode
  • Serverllama-server
  • Port47474

Solid boxes are processes. Cylinders are stores. The dotted line is the only place the host and guest exchange privileged calls — everything else is HTTP.

Quickstart

Boot, pull a model, point your editor.

~/projects/iron-oxide — zsh 
$ ironoxide up
✓ golden image ironoxide.raw (1.2GB)
✓ microVM up in 812ms
✓ GPU passthrough Metal <- Vulkan
✓ llama-server listening on :47474

$ ironoxide pull qwen2.5-coder-32b
  pulling      ████████████████████  4.2GB / 4.2GB
✓ verified sha256 a1c4…9f2b

$ export OPENAI_BASE_URL=http://localhost:47474/v1
$ opencode
